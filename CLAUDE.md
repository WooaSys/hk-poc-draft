# Project Guidelines

## Streamlit App

- Streamlit 관련 작업의 루트 디렉토리는 `app/`이다.
- 파일 경로, import, 실행 등 모두 `app/`을 기준으로 한다.

## App 작업 문서 관리

App 관련 작업 시 아래 디렉토리 구조와 규칙을 따른다.

### 디렉토리 구조

| 디렉토리 | 용도 |
|---|---|
| `01_plan/` | 작업 계획 문서 |
| `02_todo/` | 계획 수행을 위한 작업 목록 |
| `03_result/` | 작업 수행 결과 |

### 파일 네이밍

- 파일명 앞에 인덱스를 붙여 작업 단위 순서를 표시한다. (예: `01_xxx.md`, `02_xxx.md`)

### 문서 구성

각 문서의 상단에는 다음 항목을 포함한다:

- **제목**: 문서 제목
- **작성시간**: 문서 작성 시점
- **주제**: 작업 주제에 대한 간략한 요약

### 문서 간 참조 규칙

- **todo 문서**: 기반이 되는 plan 문서명을 명시한다.
- **result 문서**: 기반이 되는 plan 문서명과 todo 문서명을 명시한다.

## wemeet 데이터 관리

- `doc/wemeet/`는 고객 원본 데이터를 재가공한 app용 데이터 저장소이다.
- 해당 디렉토리에 결과 파일을 생성할 때는 `doc/wemeet/index.md`에 파일 정보를 기록한다.

## App 데이터 참조 규칙

- app에서 사용하는 정적 데이터는 `app/db/`에 복사하여 그곳을 참조한다.
- 프로젝트 루트의 `raw_data/`, `00_draft/`, `doc/wemeet/` 등을 직접 참조하지 않는다.

## 코드 파일 작성 규칙

- app 작업에 사용되는 코드 파일(.py) 작성 시 상단에 docstring으로 요약 정보를 작성한다. (목적, 입출력, 사용법)

## 작업 진행 규칙

- 구현 작업에 바로 착수하지 않는다. plan/todo 문서를 작성한 후 사용자에게 확인을 받고 진행한다.

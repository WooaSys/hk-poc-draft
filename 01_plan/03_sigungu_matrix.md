# 시군구 간 비용행렬 생성

- **작성시간**: 2026-04-02
- **주제**: 시군구 × 시군구 OSRM 기반 거리/시간 비용행렬 생성

## 요약

로컬 OSRM 서버를 이용하여 육지 247개 시군구 간 상호 거리/시간을 산출한다. 권역 연결성 분석, 고립/경계 시군구 판별, 자연스러운 권역 형태 검증, 경로 최적화 시뮬레이션에 활용.

## 배경

- RDC ↔ 시군구 행렬(02_cost_matrix)은 RDC 기준 비용만 있음
- 시군구 간 행렬이 있어야:
  - 같은 권역 내 시군구끼리 연결되어 있는지 (권역 연결성)
  - 주변 시군구와 거리가 먼 고립 시군구 발견
  - 인접 권역 경계에 있는 시군구 판별
  - 배송 순회 경로 시뮬레이션

## 입력 데이터

| 데이터 | 파일 | 비고 |
|---|---|---|
| 시군구 위치 (247개) | `doc/wemeet/sigungu_master.csv` | 제주 제외 |

## 출력

파일: `doc/wemeet/sigungu_matrix.csv`

| 컬럼 | 설명 |
|---|---|
| `from_code` | 출발 시군구 코드 (sub_region_code) |
| `from_name` | 출발 시군구명 (광역도시 + 시군구) |
| `to_code` | 도착 시군구 코드 |
| `to_name` | 도착 시군구명 |
| `duration_min` | 소요시간 (분) |
| `distance_km` | 거리 (km) |

- 총 행 수: 247 × 246 = 60,762행 (자기 자신 제외)
- 방향성 있음 (A→B ≠ B→A일 수 있음)

## 구현 방식

1. 시군구 247개 좌표 로드 (제주 제외)
2. OSRM table API 활용 — 다대다 행렬 한 번에 조회 가능
3. duration/distance 추출
4. long format CSV 저장

## 기술 스택

- Python (requests, pandas)
- OSRM table API (`/table/v1/driving/`)
- 기존 `cost_matrix.py` 모듈 확장 또는 별도 모듈

## 주의사항

- OSRM table API는 좌표 수 제한이 있을 수 있음 → 필요 시 배치 분할
- 247개 좌표 한 번에 보내면 약 60K 셀 → 로컬 서버이므로 가능할 것으로 예상
- 제주 제외

# VROOM 배차 시뮬레이션 및 비용 산출

- **작성시간**: 2026-04-03
- **주제**: 현재 상태 및 재할당 시나리오별 배차 후 비용 산출

---

## 1. 목적

현재 배정과 재할당 시나리오를 대상으로 OSRM+VROOM을 이용한 배차 시뮬레이션을 수행하고, 2025년 11월 배송 단가표 기준으로 실제 운송 비용을 산출한다.

---

## 2. 차량 대수 산정 근거 — 왜 3.5T 기준인가

### 검토 과정

RDC별 차량 현황(`14_RDC별_차량현황_추정.md`)에서 차량 대수 / 수요 비율을 역산하면:

| RDC | 3.5T 일평균 | 3.5T 수요 | 비율 |
|---|---:|---:|---:|
| 평택 | 41.3 | 84.6 | 0.49 |
| 칠곡 | 27.8 | 53.0 | 0.52 |
| 계룡 | 21.5 | 35.2 | 0.61 |
| 제천 | 11.8 | 14.0 | 0.84 |

RDC마다 비율이 다름 (적재 효율, 배송 밀도, 회전 수 차이). 재배정 후에도 같은 비율이 유지되는지 불확실.

### 결정

**3.5T 수요(`daily_demand_3_5t`)를 직접 차량 수로 사용.**

이유:
1. 처음부터 3.5T 기준으로 일관되게 분석
2. 재배정 후 수요 변화 → 차량 수 자동 조정
3. RDC별 비율 차이를 고려할 필요 없이 단순/일관적
4. 역산 비율은 현재 운영 패턴에 종속적, 재배정 후 적용 부적절

### VROOM 투입 방식

- **차량 수**: RDC별 `ceil(Σ daily_demand_3_5t)` — 해당 RDC에 배정된 시군구의 수요 합산 후 올림
- **차량 capacity**: 1.0 (3.5T 1대 = 만적 100%)
- **시군구별 job**: demand = 해당 시군구의 `daily_demand_3_5t` (적재율 비율)
- VROOM이 여러 시군구를 한 트럭에 묶어서 경로 최적화
- 예: 시군구 A(0.4) + 시군구 B(0.3) + 시군구 C(0.2) = 0.9 → 트럭 1대 경로

`daily_demand_3_5t`는 "3.5T 트럭 한 대 용량의 몇 %를 사용하는가"이므로, 1건=1대가 아니라 적재율 기반으로 자연스럽게 혼적됨.

## 3. 비용 산출 기준

- **단가표**: 2025년 11월 기준 (`raw_data/3.업체 공유용_배송 단가표_*.csv`)
- **트럭 기준**: 3.5T
- **거리 기준**: OSRM distance_km → 단가표 거리구간 매핑
- **비용 계산**: 각 배차 경로의 거리 → 해당 RDC × 거리구간의 3.5T 단가 적용

---

## 4. 시뮬레이션 구조

### 입력

| 데이터 | 파일 | 비고 |
|---|---|---|
| 시군구 수요 | `app/db/sigungu_demand.csv` | daily_demand_3_5t |
| RDC 위치 | `app/db/rdc_locations.json` | |
| 시군구 좌표 | `app/db/sigungu_master.csv` | |
| 배송 단가표 | `raw_data/3.업체 공유용_배송 단가표_*.csv` | 2025년 11월 |
| 재할당 결과 | stress_test 엔진 출력 | 시나리오별 assigned_rdc |

### 실행 흐름

1. 시나리오별 RDC-시군구 배정 확인
2. 각 RDC별 차량 대수 산정 (`ceil(daily_demand_3_5t)`)
3. VROOM으로 배차 최적화 (RDC = depot, 시군구 = delivery)
4. 배차 경로별 거리 산출
5. 거리구간 × RDC × 3.5T 단가 적용 → 비용 산출

### 비교 대상

- Baseline: 현재 운영 상태
- S05 ~ S25: 재할당 시나리오별

### 산출 지표

| 지표 | 설명 |
|---|---|
| 총 운송 비용 | 전체 시나리오별 배송 비용 합계 |
| RDC별 비용 | RDC별 배송 비용 |
| 평균 경로 거리 | 차량당 평균 운행 거리 |
| 차량 utilization | 차량 적재율 |

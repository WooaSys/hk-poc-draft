# 공간 연속성 제약 추가

- **작성시간**: 2026-04-03
- **주제**: 권역 재할당 시 고립 시군구 방지를 위한 공간 연속성 제약

## 요약

현재 optimizer가 비용 최적화만 하다 보니, 주변이 다른 RDC로 배정된 상태에서 한 시군구만 고립되는 현상이 발생. 인접 시군구와 같은 RDC에 배정되도록 유도하는 제약을 추가한다.

## 문제

- 철원군(수요 0.1)이 평택에 남고, 주변 연천/포천/가평은 제천으로 이동
- 비용 최적화 관점에서는 정상이지만, 실제 운영에서는 비효율 (고립 배송)

## 접근 방법

### 인접 시군구 정의

`sigungu_matrix.csv`에서 이동시간 기준 가장 가까운 N개를 인접으로 정의.

- 임계값 방식: 이동시간 60분 이내 시군구를 인접으로 간주
- 또는 K-nearest: 가장 가까운 5개를 인접으로 간주

### 제약 방식

**소프트 제약 (penalty 방식)** — 목적함수에 고립 패널티 추가

인접 시군구와 다른 RDC에 배정되면 패널티를 부여:
- 시군구 i의 인접 시군구 k가 다른 RDC에 배정되면 penalty 부과
- penalty 강도를 조절하여 비용 최적화와 균형

이 방식이 하드 제약보다 유연하고, solver가 infeasible 되는 위험이 없음.

## 입력 데이터

- `app/db/sigungu_matrix.csv` — 시군구 간 이동시간으로 인접 판단

## 구현 위치

- `stress_test/common.py` — 인접 리스트 생성
- `stress_test/engine_cpsat.py`, `engine_scip.py` — 패널티 항 추가

## 파라미터

- `adjacency_threshold`: 인접 판단 임계값 (기본: 60분)
- `isolation_penalty`: 고립 패널티 강도 (기본: 조절 가능)
